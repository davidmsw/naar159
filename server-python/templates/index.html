    2. Aim of this guide

      Writing a Machine-To-Machine (M2M) application can be daunting. The purpose of this tutorial is to step you through the setup of a Raspberry Pi® & Camera, the PiFace Digital Module, BigPond® Mobile Wi-Fi 4G router, and Remote Virtual Machine (VM). You are free to use any of the code that Telstra provides, and please suggest bug fixes where necessary.

      Discussion and collaboration between teams is encouraged through the use of GitHub. Although it is important not to reveal your idea or solution, where there is group benefit (e.g. improvements to the collaborative code provided by Telstra) then the teams are urged to contribute. GitHub will be introduced later.

      This tutorial will help you:

      • Set up your Raspberry Pi® and Camera;
      • Set up the PiFace Digital Module;
      • Get wireless connectivity working using a BigPond® Mobile Wi-Fi 4G router;
      • Run the provided sample GPS client;
      • Set up a VM server.

      The diagram below shows the elements you will interact with during this tutorial.

      • The Raspberry Pi® is a small computer which has standard I/O ports to allow you to connect external devices that you may need for your solution. The BigPond® Mobile Wi-Fi 4G router provides wireless connectivity to transfer data. The BigPond® Mobile Wi-Fi 4G router has an inbuilt GPS chip, which during this tutorial, we will use to acquire and transmit GPS data.

      • A VM server inside Telstra's Cloud Innovation Centre running Linux, is provided to you to support your application being developed on the Raspberry Pi®. In the example application, this VM server is configured to do two things:
        1. Receive GPS co-ordinates, and team name submitted via JSON requests from the Raspberry Pi®
        2. Provide a complete list of GPS co-ordinates most recently received for each Raspberry Pi® that has reported GPS data

        These two functions make it possible for the server to provide a map illustrating the locations of each of the devices. This is provided using Google Maps, which is hosted on the M2M Challenge website.

      • GitHub is a website that allows you to share source code. Telstra has a repository on GitHub that contains the Telstra example client collaborative code and example server code.

  4. Connecting and using PiFace Digital

    1. Intro to PiFace Digital

      The PiFace Digital is a digital input/output device that connects directly to the GPIO (General Purpose Input/Output) port on the Raspiberry Pi® board. Itâ€™s capable of reading in eight digital inputs and out putting eight digital outputs that link to onboard LED. The first two outputs also switch two 20V 5A relays, which will gives a greater amount of flexibility when creating your solution.

    2. Installing and setting up the PiFace Digital

      Note: It is recommended that if you have the Raspberry Pi® Camera in your kit, that it is installed before installing PiFace Digital.

      To install PiFace Digital, we start by enabling SPI (Serial Peripheral Interface)on the Raspiberry Pi. Open LXTerminal and enter the configuration menu by entering the following command. 

      pi@raspberrypi ~ $ sudo raspi-config


      Next, select option 8: Advanced Options




      Then select option 5: Enable SPI




      Then select Yes and exit the configuration menu.


      To install, the PiFace Digital module simply sits on top of the Raspberry Pi® connecting to the GPIO ports, as shown in the picture below.

      Next we need to install the Piface Digital libraries for Python2 and Python3, enter into the LXTerminal the following command. 

      pi@raspberrypi ~ $ sudo apt-get install python{,3}-pifacedigitalio

      If prompted, press Y and Enter.

      You can use the emulator to read the inputs and operate the outputs on the board real time for debugging. Start installing the emulator by entering into the LXTerminal the following command. 

      pi@raspberrypi ~ $ sudo apt-get install python3-pifacedigital-emulator

      If prompted, press Y and enter.

      To open the emulator, either open the PiFace Emulator application on the desktop or in the LXTerminal enter the following command. 

      pi@raspberrypi ~ $ pifacedigital-emulator

      You can enable or disable manipulating inputs by selecting the Enable option at the top left of the program. You can manually switch outputs by selecting the corresponding number at the bottom of the program. You can also simulate digital inputs by pressing the buttons on the PiFace Digital Module.

    3. Inputs/Outputs

      To start using the PiFace Digital module, you can open either IDLE or IDLE 3 off the desktop for writing and editing the programming code in python2 or python3 respectably. For this example we will use python3. So open the IDLE 3 application on the desktop, and then under the File option at the top, click on New Window. You should now have two windows open. The window titled Untitled is your workspace. It is the document you will write and run your code in. The other window, titled Python Shell, is where the results and debugging information will be presented

      The guide will go through all the relevant lines in the programming code and show the finished product at the end of this section. We assume the user has some basic programming experience and python experience.

      Our first step is to import the PiFace Digital library into our workspace. So we begin by writing the line. 

      import pifacedigitalio as p

      This line will import the PiFace Digital library and from now on we can use the letter p to reference it.

      The next step is to initialize the Piface Digital Module. As we have a instance of the PiFace Digital library imported to this workspace as the letter p, we just enter the following. 

      p.init()

      Our final step is about using either the digital_read() or digital_write() functions. With both functions, we can pass two real numerical variables through them corresponding to the pin_number (values of 0 to 7) and state (either 0 or 1). They both have the format digital_read(pin_number, state) or digital_write(pin_number, state).

      For the next example we will write a digital high or 1 to the first pin out or pin 0. 

      p.digital_write(0,1)

      To run the program, click on the Run tab at the top of the window, then click on Run Module.

      Note: You will notice that this will flip the contacts on the relay and turn on the LED. The contacts on relay k0 and k1 are trigger by writing a high to output pins 0 and 1 respectively. Each output is hardwired to a corresponding LED, and when a digital high or '1' is written to the the output, its LED is also turned on.

      The following example brings together all we have learned in this section. It will loop and turn on LED0 and switch the K0 relay for one second and turn it off for one second indefinitely.

      Note: The timer library is standard in the python package. To see more about the standard packages, there is a python section in the supporting material section here. 
      import pifacedigitalio as p	#import the piface library
from time import sleep		#import the sleep function
p.init()			#initialise the piface module
while(True):
	sleep(1)		#sleep for one second
	p.digital_write(0,1)	#set the output at pin 0 to high
	sleep(1)		#sleep for one second
	p.digital_write(0,0)	#set the output at pin 0 to low

      To exit the infinite loop, press Control+C on the keyboard.

      Note: You can also write and execute commands to operate the PiFace in LXTerminal, tho it can become tedious if the code is more than a few lines.

    5. Frequently Asked Questions

      Q. How can I check whether my router still has connectivity?

      A. There are several ways to verify this including by logging into the BigPond® router website http://telstra.4g. Alternatively, you could try loading up a website.


      Q. Why is the device using the Telstra.extranet Access Point Name (APN)?

      A. We are using this APN because it provides a public IP address to your device.


      Q. Will I always be allocated the same IP Address when I connect to the network?

      A. No, the IP addresses are dynamically allocated when you attach to the network. It is likely you will often get the same address, but this should never be assumed.


      Q. One of the components of the kit is broken. What should I do?

      A. Start by heading to the support tab[insert link] and fill out the support form. Alternatively, send a email to the support team. Be sure to detail the component that isn't working and the issue you are having and we'll be in touch shortly.


      Q. What is the VM server Configuration?

      A. The VMs are provisioned with the following configuration:
      • Intel CPU (2 Cores) @ 2.27GHz
      • 2GB of RAM
      • 20GB of disk space

      Q. I am having username/password issues with the VM server. What should I do?

      A. Contact us by either filling out the support form or emailing the support team. We will reset the username/password for you.


      Q. The Raspberry Pi® power cycles or mouse does not respond properly. What's wrong?

      A. These issues can be caused by and insufficient power supply and/or large sudden power drain on the Raspberry Pi®'s USB ports. Please ensure the power supply you are using can at least support a minimum of 1A. We also do not recommend inserting the wireless modem directly into the Raspberry Pi®'s USB ports while it is powered as this can cause a sudden drain in power that can reset the Raspberry Pi®. Where possible, we recommend using a powered USB hub to attach USB peripherals to the Raspberry Pi®.


      Q. Will the Telstra University Challenge utilise Telstra's 4G network?

      A. Teams will be provided with a Telstra 4G capable USIM and BigPond® router, but teams will need to check the current Telstra coverage maps (located here) to check what coverage is in their area.


      Q. Why is GitHub being used for in the M2M challenge?

      A. GitHub is being used for the code sharing where there is a common benefit for all universities - such as improving the code provided by Telstra.


      Q. Can the Telstra GitHub account be used for managing our own University Challenge Code?

      A. No, this is public repository. If you store your code in the Telstra GitHub account, it will be visible to other people. We recommend you use your own GitHub account or an alternative tool to manage your M2M University Challenge development code. Please note that only paid Github accounts have private repositories.


      Q. The keyboard configuration is wrong. How do I fix it?

      A. The default keyboard layout with the standard Raspberry Pi® image is British layout. You will likely have to change this to a US layout for your keyboard to work properly. Please see See the following forum or watch the following video guide.


      Q. I can only connect a mouse and keyboard into the Raspberry Pi®'s USB ports - is there no third USB port for the USB modem?

      A. You will need to use a USB hub to increase the number of ports. We recommend using a powered USB hub. Another option is to remove the mouse and keyboard and connect remotely via SSH in the Raspberry Pi® while the device is connected to a network via Ethernet or wireless.
